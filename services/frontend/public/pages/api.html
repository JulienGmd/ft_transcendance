WebSocket Game API

This page describes the WebSocket API to interact with the Pong game server.

Connection

  • URL: ws(s)://<host>/api/game/ws
  • Authentication: JWT in httpOnly cookie (same as the frontend)

Client → Server Messages

  • join_local: create local game

    { type: "join_local" }

  • join_normal: join the normal queue

    { type: "join_normal" }

  • join_tournament: join the tournament queue

    { type: "join_tournament" }

  • leave_queue: leave the queue

    { type: "leave_queue" }

  • move: send a paddle move action

    { type: "move", direction: -1 | 0 | 1, isGuest?: boolean }

  • ping: ping the server

    { type: "ping" }

Server → Client Messages

Possible message types:

queue_joined, queue_left, game_found, countdown, game_start, game_sync, paddle_update, score_update, game_over, tournament_result, error, pong

Below are the main server→client message types and their payload structures. All information needed to implement a client is included here.

  • queue_joined:

    {
  type: "queue_joined",
  mode: "normal" | "tournament"
  position: number, // your position in the queue
}

  • queue_left:

    {
  type: "queue_left"
}

  • game_found:

    {
  type: "game_found",
  mode: "normal" | "tournament"
  side: "left" | "right",
  opponentName?: string,
}

  • countdown:

    {
  type: "countdown",
  seconds: number
}

  • game_sync:

    {
  type: "game_sync",
  state: {
  ball: {
    time: number // Time since last bounce/launch
    pos: Vector2D // Position at last bounce/launch
    velocity: Vector2D // Velocity at last bounce/launch
  }
  paddles: {
    left: {
      y: number
      direction: -1 | 0 | 1
    }
    right: {
      y: number
      direction: -1 | 0 | 1
    }
  }
  score: {
    left: number
    right: number
  }
}
}

  • game_over:

    {
  "type": "game_over",
}

  • error:

    {
  "type": "error",
  "message": string
}

  • pong:

    {
  "type": "pong"
}

Example Sequence

  1. Connect to the WebSocket
  2. Send { type: "join_normal" }
  3. Receive queue_joined, then game_found, then countdown, etc.
  4. During the game, send { type: "move", ... } to move the paddle

Notes

  • All commands are JSON.
  • See the source code for more details.
